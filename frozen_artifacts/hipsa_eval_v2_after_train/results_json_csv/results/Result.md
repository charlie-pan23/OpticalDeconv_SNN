## 3. Script Improvements Already Applied

The training script was improved to support a cleaner experimental workflow.

### Important script behavior

| Feature | Current behavior |
|---|---|
| Fixed split | Training, validation, and test use the same split generated by preprocessing. |
| Checkpoint selection | Uses validation accuracy to save `best_val.pth`. |
| Final test | Runs once at the end using `best_val.pth`. |
| Test-only mode | Supports `--test-only --checkpoint <path>`. |
| Resume distinction | `--resume` is for full training state; `--weights` is for loading weights only. |
| Early stopping | Supported under `training.early_stopping`. |
| Metadata | Saves split/config/args/model info into the run directory. |

### Recommended test-only command

```bash
python train/train_cifar10dvs.py \
  --config configs/config_cifar10dvs.yaml \
  --test-only \
  --checkpoint ./results/cifar10dvs/run_xxx/checkpoints/best_val.pth
```

| Metric | Exp-1 | Exp-2 | Change |
|---|---:|---:|---:|


- Therefore, simple batch-size and regularization tuning does not fully solve the generalization bottleneck.

## 8. Next Planned Experiment: Input Encoding Ablation

### Experiment 3 Goal

Test whether the binary spike input is the main accuracy bottleneck.

### Planned configuration

| Item | Value |
|---|---|
| Config | `configs/config_cifar10dvs_clipped_count3.yaml` |
| Input encoding | `clipped_count` |
| Clipping max | `3.0` |
| Batch size | train `64`, eval `128` |
| Weight decay | `0.001` |
| Dropout | `0.3` |
| Early stopping | min epochs `70`, patience `20`, min delta `0.0` |

### Expected interpretation

| Result | Interpretation |
|---|---|
| clipped_count reaches `78%–85%` | Binary input is a major accuracy bottleneck. |
| clipped_count reaches only `72%–74%` | Main bottleneck is more likely model/training/data augmentation. |
| clipped_count improves by `1%–3%` | Binary input hurts accuracy, but it is not the only bottleneck. |

### Important note

This experiment is an ablation / software upper-bound test. It does not immediately replace the binary-input HIPSA hardware-facing baseline.

## 9. Commands for Experiment 3

### Create new config

```bash
python - <<'PY'
import yaml
from pathlib import Path

src = Path('configs/config_cifar10dvs_b64_wd001_do03.yaml')
dst = Path('configs/config_cifar10dvs_clipped_count3.yaml')

cfg = yaml.safe_load(src.read_text(encoding='utf-8'))

cfg.setdefault('preprocess', {})
cfg['preprocess']['input_encoding'] = 'clipped_count'
cfg['preprocess']['clipped_count_max'] = 3.0

cfg.setdefault('training', {})
cfg['training']['batch_size_train'] = 64
cfg['training']['batch_size_eval'] = 128

cfg['training'].setdefault('early_stopping', {})
cfg['training']['early_stopping'].update({
    'enabled': True,
    'min_epochs': 70,
    'patience': 20,
    'min_delta': 0.0,
})

# Keep the same regularized setting from Exp-2.
if isinstance(cfg.get('optimizer'), dict):
    cfg['optimizer']['weight_decay'] = 0.001
elif isinstance(cfg['training'].get('optimizer'), dict):
    cfg['training']['optimizer']['weight_decay'] = 0.001
else:
    cfg['training']['weight_decay'] = 0.001

cfg.setdefault('model', {})
cfg['model']['dropout'] = 0.3

dst.write_text(yaml.safe_dump(cfg, sort_keys=False, allow_unicode=True), encoding='utf-8')
print(f'Wrote {dst}')
PY
```

### Verify config and code support

```bash
grep -nE "input_encoding|clipped_count|batch_size_train|batch_size_eval|weight_decay|dropout|early_stopping|min_epochs|patience|min_delta" \
  configs/config_cifar10dvs_clipped_count3.yaml
```

```bash
grep -R "input_encoding\|clipped_count" -n train models configs | head -50
```

If `input_encoding` or `clipped_count` only appears in config files and not in training code, the training code needs to be updated before running this experiment.

### Run training

```bash
python train/train_cifar10dvs.py --config configs/config_cifar10dvs_clipped_count3.yaml
```
